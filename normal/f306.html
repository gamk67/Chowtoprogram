Bubble sort

  • Improving the performance

    • The final passes are often unnecessary
    • Worst case occurs when smallest number is top element
    • Smallest number moves left one position during each pass
    • Quit if we don't make changes during a pass
    • Track this with variable 'changed'

  • Further improvement:

    • After first pass, largest element is in position
    • After second pass, second largest is in position
    • etc.
    • Reduce number of iterations with each pass
